## Design decisions to preserve

- `wip/` should mean abandoned or inactive experiments. Active Plan B material
  belongs in `src/`; this has already been applied to Path B.
- Metadata in Lean is a machine-local ledger. The reviewer-facing proof map is
  still leanblueprint LaTeX.
- Status claims should eventually be machine-derived from the Lean kernel
  footprint, not hand-maintained. The intended source is `Lean.collectAxioms`.
- The four categories mean:
  - `clean`: only canonical Lean axioms such as `propext`, `Classical.choice`,
    and `Quot.sound`;
  - `trusted`: explicitly allowlisted literature or finite trusted axiom;
  - `debt`: a real open capstone intended to be discharged;
  - `broken`: `sorryAx`, undeclared axiom leakage, or a node that should be
    unconditional but is not.
- Do not hand-flip `debt` to `clean` or add `\leanok` because a theorem name
  exists. Existence is not proof of clean dependency footprint.
- Do not import `GoodsteinPABlueprint` from `GoodsteinPA.lean` unless Trevor
  explicitly decides that ordinary users should see the audit ledger.
