2 Two shores: the formalized girders

The program is pitched between two shores of pre-existing, kernel-clean formalization. These nodes are green anchors we build on, not obligations we owe.

2.1 mathlib: ordinal notation and the fast-growing hierarchy

Definition 2.1
✓
#

Lean declarations

  • ONote

mathlib’s constructive ordinal notation system ONote (Cantor normal form as an inductive type), together with its normal-form predicate ONote.NF. This is the concrete, computable carrier for every ordinal \({\lt}\varepsilon _0\) used throughout the program. (mathlib also has the semantic Ordinal.CNF decomposition, but it is not in this build’s import closure; the notation system ONote is what the growth theory runs on.)

Definition 2.2
✓
#

Lean declarations

  • Ordinal.veblen

The two-argument Veblen function \(\varphi \), from which mathlib defines the \(\varepsilon \)-function \(\varepsilon _o = \texttt{veblen}\, 1\, o\) and hence the ceiling \(\varepsilon _0\). This is the semantic ordinal boundary that PA’s proof-theoretic ordinal must equal; ONote notations are its below-\(\varepsilon _0\) codes.

The fast-growing hierarchy \(f_o : \mathbb {N}\to \mathbb {N}\) on ordinal notations (\(f_0 = \mathrm{succ}\), \(f_{o+1}(n) = f_o^{[n]}(n)\), \(f_\lambda (n) = f_{\lambda [n]}(n)\) at limits via fundamentalSequence). mathlib supplies the definition and small values but none of the growth theory (expansiveness, monotonicity, domination); those are developed in this program’s Hardy port and Domination modules.

2.2 Foundation: first-order logic and incompleteness

Theorem 2.4
✓
#

Lean declarations

  • LO.FirstOrder.hauptsatz

Foundation’s finitary first-order Gentzen Hauptsatz (cut-elimination), LO.FirstOrder.hauptsatz (\(=\texttt{Hauptsatz.main}\)). It is proved and kernel-clean in Foundation, and is now imported into this build (via the blueprint root), so this node is a real green shore. It is listed as a shore because the infinitary, \(\varepsilon _0\)-ordinal-assigned analogue of exactly this theorem is the crown gap (Thm 6.1); the finitary Hauptsatz is the template, not the summit.

  • LO.FirstOrder.Arithmetic.PA_delta1Definable

\(\mathsf{PA}\) is \(\Delta _1\)-definable (\(\mathsf{PA}.\Delta _1\)). Foundation formerly posited this as a bare axiom; it now proves it as a real instance (InductionSchemeDelta1.lean), which is exactly why this program’s Gödel-II surfacing is axiom-clean. This discharge was a genuine upstream gift (gotrevor/Foundation@e6e1ad1).
